# Time

## Segment Type

`time`

## What It Shows

The `time` segment can show two different timestamps:

- `.LastDate`
  - the time the previous command finished
  - generated by prompto when the prompt is built
- `.CurrentDate`
  - the time the current command started
  - generated by the shell when the prompt is shown

That difference is intentional.

## Default Template

```template
{{ .LastDate | date .Format }}
```

## Basic Configuration

```yaml
prompt:
  - segments: [time]

time:
  type: time
  style: plain
  foreground: "#007ACC"
  options:
    time_format: "15:04:05"
```

## Available Fields

- `.Format`
  - the resolved `time_format`
- `.LastDate`
  - the previous command finish time
- `.CurrentDate`
  - the current command start time, already formatted for display

In practice, that means these two forms are both valid, but they do different things:

```template
{{ .LastDate | date .Format }}
```

```template
{{ .CurrentDate }}
```

## Behavior By Prompt Type

In `prompt` and `rprompt`:

- `.LastDate` shows when the previous command finished
- `.CurrentDate` effectively shows the same thing, because there is no current command yet

In `transient` and `rtransient`:

- `.LastDate` still shows when the previous command finished
- `.CurrentDate` shows when the current command started

## Example: `sleep 3m`

Assume this timeline:

1. Your previous command finishes at `10:00:00`
2. You wait on the prompt for two minutes
3. At `10:02:00` you press Enter on `sleep 3m`
4. The command finishes at `10:05:00`

While you are typing `sleep 3m`, this is still a normal prompt:

```text
.LastDate    -> 10:00:00
.CurrentDate -> 10:00:00
```

When you press Enter, the transient prompt is shown for the command you just started:

```text
.LastDate    -> 10:00:00
.CurrentDate -> 10:02:00
```

When `sleep 3m` finishes, the next normal prompt is shown:

```text
.LastDate    -> 10:05:00
.CurrentDate -> 10:05:00
```

## Choosing Between `.LastDate` And `.CurrentDate`

Use `.LastDate` when you want the time the previous command finished.

Use `.CurrentDate` when you want transient prompts to show the time the current command started.

Example:

```yaml
time:
  type: time
  template: " {{ .CurrentDate }} "
  options:
    time_format: "15:04:05"
```

## `time_format`

`time_format` is validated when the config is loaded.
If the format is not supported, the config fails to load.
There is no silent fallback.

## Supported Format Parts

These parts are supported for both `.LastDate` and `.CurrentDate`:

- year: `2006`, `06`
- month: `January`, `Jan`, `01`
- day: `02`, `_2`, `2`
- weekday: `Monday`, `Mon`
- hour: `15`, `03`, `3`
- minute: `04`
- second: `05`
- am/pm: `PM`
- timezone name: `MST`
- numeric offset: `-0700`

## Literal Text

Literal text is allowed inside `time_format`.

Examples that are valid:

- `Mon | 15:04:05`
- `15:04 (Mon)`
- `2 Jan, Monday`
- `3:04:05 PM | Monday`

Prompt styling is not allowed inside `time_format`.

This is invalid:

```yaml
options:
  time_format: "Monday <#fff>at</> 3:04 PM"
```

Put styling in the template instead:

```yaml
time:
  type: time
  template: ' {{ .LastDate | date "Monday" }} <#fff>at</> {{ .LastDate | date "3:04 PM" }} '
```

## Supported Built-In Format Names

These built-in names are supported:

- `Layout`
- `ANSIC`
- `UnixDate`
- `RubyDate`
- `RFC822`
- `RFC822Z`
- `RFC850`
- `RFC1123`
- `RFC1123Z`
- `Kitchen`
- `Stamp`
- `DateTime`
- `DateOnly`
- `TimeOnly`

## Unsupported Formats

These are rejected by config validation:

- `1`, `4`, `5`
- `pm`
- fractional seconds such as `.000`, `.999999`, `.999999999`
- timezone forms such as `-07`, `-07:00`, `Z0700`, `Z07:00`
- built-in names such as `RFC3339`, `RFC3339Nano`, `StampMilli`, `StampMicro`, `StampNano`
- prompt styling such as `<#fff>...`

## Examples

Display the previous command finish time:

```yaml
time:
  type: time
  template: " {{ .LastDate | date .Format }} "
  options:
    time_format: "15:04:05"
```

Display the current command start time in a transient prompt:

```yaml
transient:
  - segments: [time]

time:
  type: time
  template: " {{ .CurrentDate }} "
  options:
    time_format: "15:04:05"
```

Display the previous command finish time in UTC as a second value:

```template
{{ .LastDate | date .Format }} {{ dateInZone "15:04Z" .LastDate "UTC" }}
```
